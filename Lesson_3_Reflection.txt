When would you want to use a remote repository rather than keeping all your work
local?
  You want to use a remote repository rather than keeping all of your work local if you would like to store your files and commits in the cloud to allow for easy backups and collaboration.

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?
  You might want to manually pull changes because there is a good chance you want some of the stuff that is on GitHub to stay only on GitHub and not copy it to your local repository.
